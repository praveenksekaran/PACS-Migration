# dicomweb-react-viewer — Elastic Beanstalk Deployment Plan

## Architecture

```
Browser
    │  HTTP :80
    ▼
EB nginx (port 80)
    │  reverse proxy to port 8080
    ▼
Node.js Fastify server  (server.js — inside dicomweb-react-viewer/)
    ├── GET /app-config.js  → dynamically generated from API_URL env var
    ├── GET /assets/*       → pre-built React/Vite static files (dist/)
    └── GET /*              → SPA fallback (dist/index.html)
```

**What gets deployed:** `dicomweb-react-viewer/` — contains both the Fastify server (`server.js`) and the pre-built static files (`dist/`)

## Phase 3 — Create Deployment Zip

EB runs `npm install --omit=dev` during deploy so exclude `node_modules/`.
Also exclude `src/`, `e2e/`, `public/` (source files not needed at runtime).

```bash
cd dicomweb-react-viewer
zip -r ../dicomweb-viewer-v1.zip . \
  --exclude "node_modules/*" \
  --exclude "src/*" \
  --exclude "e2e/*" \
  --exclude "playwright-report/*" \
  --exclude "*.ts" \
  --exclude "*.tsx"
```

The zip includes: `server.js`, `package.json`, `package-lock.json`, `dist/`, `public/` (runtime app-config.js), `index.html`

**IMPORTANT — Windows only:** PowerShell `Compress-Archive` uses backslash paths which Linux unzip rejects. Use Python instead:

```python
# make-zip.py
import zipfile, os
src  = r"C:\Praveen\Projects\dicomweb-pacs-src\dicomweb-react-viewer"
dest = r"C:\Praveen\Projects\dicomweb-pacs-src\dicomweb-viewer-v1.zip"
include_roots = {"server.js", "package.json", "package-lock.json", "dist", "public", "index.html"}
with zipfile.ZipFile(dest, "w", zipfile.ZIP_DEFLATED) as zf:
    for root, dirs, files in os.walk(src):
        rel_root = os.path.relpath(root, src)
        top = rel_root.split(os.sep)[0]
        if top not in include_roots and rel_root != ".":
            dirs[:] = []; continue
        for fname in files:
            abs_path = os.path.join(root, fname)
            arcname = os.path.relpath(abs_path, src).replace(os.sep, "/")
            zf.write(abs_path, arcname)
```
```bash
python make-zip.py
```

## Phase 4 — Upload to S3 & Create App Version

```bash
# Upload zip to S3 — must be same region as EB (ap-southeast-2)
# Bucket: dicomweb-eb-deployments (created in ap-southeast-2)
aws s3 cp ../dicomweb-viewer-v1.zip \
  s3://dicomweb-eb-deployments/dicomweb-viewer-v1.zip \
  --region ap-southeast-2

# Register the version with EB
aws elasticbeanstalk create-application-version \
  --application-name dicomweb-viewer \
  --version-label v1 \
  --source-bundle S3Bucket=dicomweb-eb-deployments,S3Key=dicomweb-viewer-v1.zip \
  --region ap-southeast-2
```

## Phase 5 — Create EB Environment

Get the exact Node.js 20 solution stack name first:
```bash
aws elasticbeanstalk list-available-solution-stacks \
  --region ap-southeast-2 \
  --query "SolutionStacks[?contains(@,'Node.js 20')]" \
  --output table
```

Then create the environment:
```bash
aws elasticbeanstalk create-environment \
  --application-name dicomweb-viewer \
  --environment-name dicomweb-viewer-prod \
  --solution-stack-name "<NODE_20_STACK_FROM_ABOVE>" \
  --version-label v1 \
  --option-settings \
    Namespace=aws:autoscaling:launchconfiguration,OptionName=InstanceType,Value=t3.micro \
    Namespace=aws:elasticbeanstalk:application:environment,OptionName=API_URL,Value=http://3.27.26.186:5001 \
  --region ap-southeast-2
```

**Key:** EB automatically sets `PORT=8080` — `server.js` reads `process.env.PORT` so no extra config needed.

## Phase 7 — Update API CORS

Once the EB URL is known, update the API's `viewerOrigin` to allow the viewer domain:

```bash
# Register task definition rev 3 with updated viewerOrigin
MSYS_NO_PATHCONV=1 aws ecs register-task-definition \
  --region ap-southeast-2 \
  --family dicomweb-api \
  ...
  --container-definitions '[{
    "environment": [
      {"name": "NODE_CONFIG", "value": "{...\"viewerOrigin\":\"http://<EB_CNAME>\"}"}
    ]
  }]'

# Force redeploy
aws ecs update-service \
  --cluster dicomweb-cluster \
  --service dicomweb-api \
  --task-definition dicomweb-api:3 \
  --force-new-deployment \
  --region ap-southeast-2
```

## Redeployment (after code changes)

```bash
# 1. Rebuild
cd dicomweb-react-viewer && npm run build

# 2. Re-zip
zip -r ../dicomweb-viewer-v2.zip . \
  --exclude "node_modules/*" --exclude "src/*" --exclude "e2e/*" \
  --exclude "playwright-report/*" --exclude "*.ts" --exclude "*.tsx"

# 3. Upload + register
aws s3 cp ../dicomweb-viewer-v2.zip \
  s3://dicomweb-data/eb-deployments/dicomweb-viewer-v2.zip \
  --region ap-south-2

aws elasticbeanstalk create-application-version \
  --application-name dicomweb-viewer --version-label v2 \
  --source-bundle S3Bucket=dicomweb-data,S3Key=eb-deployments/dicomweb-viewer-v2.zip \
  --region ap-southeast-2

# 4. Deploy
aws elasticbeanstalk update-environment \
  --environment-name dicomweb-viewer-prod \
  --version-label v2 \
  --region ap-southeast-2
```

## Notes

| Item | Detail |
|---|---|
| Port | `server.js` reads `process.env.PORT` — EB sets this to 8080 automatically |
| API URL | Set via `API_URL` env var in EB. Currently: `http://3.27.26.186:5001` |
| COOP/COEP headers | Set in `server.js` — MPR/SharedArrayBuffer works out of the box |
| `app-config.js` | Dynamically served by server — change `API_URL` in EB env vars without rebuilding |
| Build output | `dist/` inside `dicomweb-react-viewer/` — must build before zipping |
| S3 upload region | `ap-south-2` (where the bucket is) — EB reads it from there in `ap-southeast-2` |
