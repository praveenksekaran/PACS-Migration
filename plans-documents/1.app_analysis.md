# dicomweb-pacs Application Analysis

## Application Overview

This is a self-contained **DICOM PACS (Picture Archiving and Communication System)** — a single Node.js package that includes both a backend server and a pre-built web viewer.

---

## Architecture Diagram

```
DICOM Devices (CT/MRI)
        │ DIMSE Protocol (C-Store/C-Find/C-Echo)
        │ Port 8888
┌───────▼────────────────────────────────┐
│  DIMSE Server (dicom-dimse-native)     │
│  - Store SCP (receives images)         │
│  - Find SCU (queries metadata)         │
└───────┬────────────────────────────────┘
        │
┌───────▼────────────────────────────────┐
│  File System Storage (./data/)         │
│  StudyUID/ → InstanceUID (raw DICOM)  │
└───────┬────────────────────────────────┘
        │
┌───────▼────────────────────────────────┐
│  Fastify Web Server (Port 5001)        │
│  - QIDO-RS  (search/query)            │
│  - WADO-RS  (retrieve images)         │
│  - WADO-URI (legacy retrieval)        │
│  - Static files (serves OHIF viewer)  │
└───────┬────────────────────────────────┘
        │ HTTP/REST Port 5001
┌───────▼────────────────────────────────┐
│  OHIF Viewer v3.10.1 (browser SPA)    │

│  - Image viewer + MPR (VTK.js/WASM)  │
│  - Annotations, measurements          │
└────────────────────────────────────────┘
```

## Two Main Components

### 1. Server (`src/`)

| File | Purpose |
|------|---------|
| `src/app.js` | Main entry point — starts Fastify server + DIMSE |
| `src/routes/routes.js` | DICOMweb REST API endpoints |
| `src/utils.js` | DIMSE operations, DICOM tag processing, file utilities |
| `src/storescu.js` | Import tool — pushes files from `./import/` to the server |

**Key design points:**
- **Fastify** is the web framework (fast, minimal)
- **No traditional database** — metadata is extracted on-demand from DICOM files stored on disk
- DIMSE and HTTP servers run concurrently in the same process
- Graceful shutdown via `close-with-grace`

**DICOMweb APIs implemented:**
- `GET /rs/studies` — QIDO-RS study search
- `GET /rs/studies/{uid}/series` — series search
- `GET /rs/studies/{uid}/series/{uid}/instances` — instance search
- `GET /rs/studies/{uid}/series/{uid}/instances/{uid}/frames/{n}` — pixel data
- `GET /wadouri?...` — legacy WADO-URI

### 2. Viewer (`public/`)

- **OHIF Viewer v3.10.1** — pre-built React SPA, no build step required
- Configured via `public/app-config.js` to point at the local DICOMweb API (`qidoRoot: "rs"`, `wadoRoot: "rs"`)
- Supports MPR via VTK.js WASM modules
- All unmatched routes serve `index.html` to support SPA client-side routing

## Key Configuration (`config/default.js`)

| Setting | Default | Purpose |
|---------|---------|---------|
| `source.port` | `8888` | DIMSE listener port |
| `webserverPort` | `5001` | HTTP server port |
| `storagePath` | `./data` | Where DICOM files are stored |
| `source.aet` | `DICOMWEB_PACS` | DICOM Application Entity Title |
| `peers` | `[{...}]` | Remote PACS servers to connect to |
| `transferSyntax` | `1.2.840.10008.1.2` | Default transfer syntax (Implicit VR) |
| `qidoMinChars` | `0` | Minimum characters required for a query |
| `qidoAppendWildcard` | `true` | Auto-append wildcard to queries |
| `permissiveMode` | `true` | Allow any AET (unrestricted) |
| `verboseLogging` | `false` | DIMSE packet-level logging |

## DICOM Standards Implemented

| Standard | Description |
|----------|-------------|
| **QIDO-RS** | Query based on ID for DICOM Objects by RESTful Services |
| **WADO-RS** | Web Access to DICOM Objects by RESTful Services |
| **WADO-URI** | Web Access to DICOM Objects by URI (legacy) |
| **STOW-RS** | Store Over the Web (via DIMSE Store SCP) |
| **DIMSE C-Store** | Receive DICOM files from modalities |
| **DIMSE C-Find** | Query DICOM metadata |
| **DIMSE C-Echo** | Connectivity verification (ping) |

---

## Storage Model

- **Type**: File-system based (no traditional database)
- **Structure**: `data/{StudyInstanceUID}/{SOPInstanceUID}` (raw DICOM files)
- **Querying**: DICOM tags extracted on-demand from files via C-Find
- **Import**: Drop files in `./import/` and run `npm run import`, or use any DICOM C-Store SCU

---

## Running the Application

```bash
# Install dependencies
npm install

# Start the server (HTTP port 5001, DIMSE port 8888)
npm start

# Import DICOM files from ./import directory
npm run import

# Access the viewer
open http://localhost:5001
```

## Key Technical Decisions

| Aspect | Choice | Rationale |
|--------|--------|-----------|
| Backend framework | Fastify | High performance, minimal overhead |
| DICOM networking | dicom-dimse-native (native binding) | Full DIMSE protocol support |
| Storage | File-system | Simple, no DB maintenance required |
| Viewer | OHIF v3.10.1 (pre-built) | Industry-standard, feature-rich, no build step |
| Metadata source | On-demand file parsing | No indexing overhead, always consistent |

---

## Summary

This is a **minimal but complete PACS** — roughly 426 lines of backend code implementing the full DICOMweb standard, paired with the full-featured OHIF viewer. It uses the filesystem as its "database," making it simple to deploy with no external dependencies beyond Node.js.
