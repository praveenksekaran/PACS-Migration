# Behaviours — dicomweb-react-viewer (BDD Requirements)

> Written in Gherkin syntax (Feature / Scenario / Given / When / Then).
> Each scenario maps directly to a Jest + Playwright or Cucumber automated test.
> Organised by the 7 implementation phases from `migrationplan.md`.

---

## Feature 1: Application Startup and Configuration

> Covers Phase 1 — Project Scaffold

---

### Scenario 1.1: Application loads with API URL from server-injected config

```gherkin
Given the viewer server has injected window.config = { apiUrl: "http://localhost:5001" }
When the application initialises
Then the API base URL used for all requests is "http://localhost:5001"
And no hardcoded localhost URL is embedded in any component
```

---

### Scenario 1.2: Application falls back to empty base URL in development mode

```gherkin
Given window.config is not present on the page
When the application initialises
Then the API base URL resolves to an empty string ""
And all API requests are made to relative paths (e.g. "/rs/studies")
And the Vite dev proxy forwards those requests to "http://localhost:5001"
```

---

### Scenario 1.3: cornerstone3D initialises before any viewport is rendered

```gherkin
Given the application HTML has loaded
When the React root mounts
Then cornerstoneCore.init() is called before any CornerstoneViewport component renders
And cornerstoneToolsInit() is called before any ToolGroup is created
And the wadouri image loader is registered under the "wadouri:" URI scheme
And the wadors image loader is registered under the "wadors:" URI scheme
```

---

### Scenario 1.4: Application shell renders the main layout regions

```gherkin
Given the application has initialised successfully
When the Shell component renders
Then a menu bar is visible at the top of the screen
And a toolbar is visible below the menu bar
And a study tree panel is visible on the left side
And a viewer canvas area is visible on the right side
And a status bar is visible at the bottom of the screen
```

---

### Scenario 1.5: SharedArrayBuffer is available for MPR volume rendering

```gherkin
Given the viewer server sets the response headers:
  | Header                        | Value          |
  | Cross-Origin-Embedder-Policy  | require-corp   |
  | Cross-Origin-Opener-Policy    | same-origin    |
  | Cross-Origin-Resource-Policy  | same-site      |
When the page loads in a browser
Then typeof SharedArrayBuffer is "function"
And cornerstone3D volume rendering is available without errors
```

### Scenario 2.5: User searches with empty search text (retrieve all)

```gherkin
  | imageId |

  | wadouri:http://localhost:5001/wadouri?studyUID=1.2.3&seriesUID=4.5.6&objectUID=7.8.11  |
```

```gherkin
And the canvas is not blank or empty
```
