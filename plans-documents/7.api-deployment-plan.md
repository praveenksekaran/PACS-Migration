# dicomweb-api — ECS Deployment Plan

## Architecture

```
Internet
    │  HTTPS/HTTP :5001
    ▼
EC2 t2.micro (free tier)
└── ECS Container (dicomweb-api)
        ├── QIDO ──► RDS PostgreSQL  (ap-southeast-2)
        └── WADO ──► S3 dicomweb-data (ap-south-2, cross-region)
```

**Approach:** ECS on EC2 launch type
- ECS orchestration: free
- EC2 t2.micro: free tier (750 hrs/month, 12 months)
- ECR image storage: 500 MB free/month
- No load balancer (saves cost) — access via EC2 public IP/DNS directly

**Region:** `ap-southeast-2` (Sydney) — same as RDS

---

## Prerequisites

- AWS CLI installed and configured (`aws configure`)
- Docker Desktop running
- Node.js 20+ installed

## Phase 1 — Local Build & Test

### 1.1 Install clean dependencies
```bash
cd dicomweb-api
npm install
```

### 1.2 Build Docker image
```bash
docker build -t dicomweb-api .
```

### 1.3 Run locally to verify startup
```bash
docker run --rm -p 5001:5001 \
  -e NODE_CONFIG='{"db":{"host":"dicomwebmig.clacs4oiaso2.ap-southeast-2.rds.amazonaws.com","port":5432,"database":"dicomweb","user":"postgres","password":"YOUR_PASSWORD"},"aws":{"region":"ap-south-2","dataBucket":"dicomweb-data"},"viewerOrigin":"*"}' \
  dicomweb-api
```

Test: `curl http://localhost:5001/rs/studies` — should return JSON array.

## Phase 2 — Push Image to ECR

### 2.1 Create ECR repository
```bash
aws ecr create-repository \
  --repository-name dicomweb-api \
  --region ap-southeast-2
```
Note the `repositoryUri` from the output (e.g. `982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api`).

### 2.2 Authenticate Docker to ECR
```bash
aws ecr get-login-password --region ap-southeast-2 \
  | docker login --username AWS --password-stdin \
    982479883135.dkr.ecr.ap-southeast-2.amazonaws.com
```

### 2.3 Tag and push image
```bash
docker tag dicomweb-api:latest \
  982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api:latest

docker push \
  982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api:latest
```

## Phase 3 — IAM Roles

Two roles are needed:

### 3.1 ECS Task Execution Role (lets ECS pull the image from ECR)
This role already exists in most AWS accounts as `ecsTaskExecutionRole`.
Check:
```bash
aws iam get-role --role-name ecsTaskExecutionRole
```
If it doesn't exist:
```bash
aws iam create-role \
  --role-name ecsTaskExecutionRole \
  --assume-role-policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow","Principal":{"Service":"ecs-tasks.amazonaws.com"},"Action":"sts:AssumeRole"}]
  }'

aws iam attach-role-policy \
  --role-name ecsTaskExecutionRole \
  --policy-arn arn:aws:iam::aws:policy/service-role/AmazonECSTaskExecutionRolePolicy
```

### 3.2 ECS Task Role (lets the container read S3)
```bash
aws iam create-role \
  --role-name dicomweb-api-task-role \
  --assume-role-policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow","Principal":{"Service":"ecs-tasks.amazonaws.com"},"Action":"sts:AssumeRole"}]
  }'

aws iam put-role-policy \
  --role-name dicomweb-api-task-role \
  --policy-name s3-read-dicomweb-data \
  --policy-document '{
    "Version":"2012-10-17",
    "Statement":[{
      "Effect":"Allow",
      "Action":["s3:GetObject"],
      "Resource":"arn:aws:s3:::dicomweb-data/*"
    }]
  }'
```

## Phase 4 — ECS Cluster (EC2 Launch Type)

### 4.1 Create cluster
```bash
aws ecs create-cluster \
  --cluster-name dicomweb-cluster \
  --region ap-southeast-2
```

### 4.2 Create EC2 instance to join the cluster

#### a) Find the latest ECS-optimised AMI for ap-southeast-2
```bash
aws ssm get-parameters \
  --names /aws/service/ecs/optimized-ami/amazon-linux-2/recommended \
  --region ap-southeast-2 \
  --query "Parameters[0].Value" --output text
```
Note the `image_id` from the JSON output.

#### b) Create a security group
```bash
aws ec2 create-security-group \
  --group-name dicomweb-api-sg \
  --description "dicomweb-api ECS host" \
  --region ap-southeast-2
```
Note the `GroupId`.

Allow inbound port 5001 (API) and 22 (SSH for troubleshooting):
```bash
# Replace sg-XXXXXXXX with your GroupId
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp --port 5001 --cidr 0.0.0.0/0 \
  --region ap-southeast-2

aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp --port 22 --cidr 0.0.0.0/0 \
  --region ap-southeast-2
```

#### c) Create EC2 instance profile for ECS
```bash
aws iam create-role \
  --role-name ecsInstanceRole \
  --assume-role-policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow","Principal":{"Service":"ec2.amazonaws.com"},"Action":"sts:AssumeRole"}]
  }'

aws iam attach-role-policy \
  --role-name ecsInstanceRole \
  --policy-arn arn:aws:iam::aws:policy/service-role/AmazonEC2ContainerServiceforEC2Role

aws iam create-instance-profile --instance-profile-name ecsInstanceProfile
aws iam add-role-to-instance-profile \
  --instance-profile-name ecsInstanceProfile \
  --role-name ecsInstanceRole
```

#### d) Launch t2.micro EC2 and register to cluster
The user-data script tells the instance which ECS cluster to join.

```bash
aws ec2 run-instances \
  --image-id <AMI_ID_FROM_STEP_a> \
  --instance-type t2.micro \
  --iam-instance-profile Name=ecsInstanceProfile \
  --security-group-ids sg-XXXXXXXX \
  --user-data '#!/bin/bash
echo ECS_CLUSTER=dicomweb-cluster >> /etc/ecs/ecs.config' \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=dicomweb-ecs-host}]' \
  --region ap-southeast-2
```

Wait 1-2 minutes, then verify the instance registered:
```bash
aws ecs list-container-instances \
  --cluster dicomweb-cluster \
  --region ap-southeast-2
```

## Phase 5 — Task Definition

Create file `task-definition.json`:

```json
{
  "family": "dicomweb-api",
  "taskRoleArn": "arn:aws:iam::982479883135:role/dicomweb-api-task-role",
  "executionRoleArn": "arn:aws:iam::982479883135:role/ecsTaskExecutionRole",
  "networkMode": "bridge",
  "containerDefinitions": [
    {
      "name": "dicomweb-api",
      "image": "982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api:latest",
      "memory": 512,
      "cpu": 256,
      "essential": true,
      "portMappings": [
        {
          "containerPort": 5001,
          "hostPort": 5001,
          "protocol": "tcp"
        }
      ],
      "environment": [
        {
          "name": "NODE_CONFIG",
          "value": "{\"db\":{\"host\":\"dicomwebmig.clacs4oiaso2.ap-southeast-2.rds.amazonaws.com\",\"port\":5432,\"database\":\"dicomweb\",\"user\":\"postgres\",\"password\":\"YOUR_PASSWORD\"},\"aws\":{\"region\":\"ap-south-2\",\"dataBucket\":\"dicomweb-data\"},\"viewerOrigin\":\"*\"}"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/dicomweb-api",
          "awslogs-region": "ap-southeast-2",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ]
}
```

Register the task definition:
```bash
aws ecs register-task-definition \
  --cli-input-json file://task-definition.json \
  --region ap-southeast-2
```

Create the CloudWatch log group:
```bash
aws logs create-log-group \
  --log-group-name /ecs/dicomweb-api \
  --region ap-southeast-2
```

## Phase 6 — ECS Service

```bash
aws ecs create-service \
  --cluster dicomweb-cluster \
  --service-name dicomweb-api-service \
  --task-definition dicomweb-api \
  --desired-count 1 \
  --launch-type EC2 \
  --region ap-southeast-2
```

Verify the task is running:
```bash
aws ecs list-tasks \
  --cluster dicomweb-cluster \
  --region ap-southeast-2
```

## Phase 7 — Get Public URL & Test

Get the EC2 instance public DNS:
```bash
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=dicomweb-ecs-host" \
  --query "Reservations[0].Instances[0].PublicDnsName" \
  --output text \
  --region ap-southeast-2
```

Test endpoints:
```bash
# QIDO — list studies
curl http://<EC2_PUBLIC_DNS>:5001/rs/studies

# WADO metadata
curl http://<EC2_PUBLIC_DNS>:5001/rs/studies/<studyUID>/series
```

Update `viewerOrigin` in config (or NODE_CONFIG env var) to allow the viewer's domain once deployed.

## Phase 8 — Update Viewer Config

In `dicomweb-viewer/public/config.js`, update the API URL:
```js
window.config = {
  dataSources: [{
    configuration: {
      qidoRoot: 'http://<EC2_PUBLIC_DNS>:5001/rs',
      wadoRoot: 'http://<EC2_PUBLIC_DNS>:5001/rs',
      wadoUriRoot: 'http://<EC2_PUBLIC_DNS>:5001/wadouri',
    }
  }]
};
```

## Redeployment (after code changes)

```bash
# 1. Rebuild and push new image
docker build -t dicomweb-api .
docker tag dicomweb-api:latest 982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api:latest
docker push 982479883135.dkr.ecr.ap-southeast-2.amazonaws.com/dicomweb-api:latest

# 2. Force ECS to pull the new image
aws ecs update-service \
  --cluster dicomweb-cluster \
  --service dicomweb-api-service \
  --force-new-deployment \
  --region ap-southeast-2
```
