## 2. Architecture Decision

### Chosen Stack

| Layer | Choice | Why |
|---|---|---|
| Build tool | **Vite 5.x** | Native ESM, HMR, WASM-friendly, essential for cornerstone3D's Web Workers |
| Language | **TypeScript 5.x** | Type safety for DICOM JSON tags; IDE completion for cornerstone3D APIs |
| UI | **React 18** | Component model maps 1:1 to WPF UserControl/ViewModel pairs |
| State | **Zustand 4.x** | Global slices replace `GalaSoft.MvvmLight.Messaging`; cross-store calls replace string-keyed Messenger events |
| DICOM rendering | **@cornerstonejs/core 2.x** | De-facto browser standard; WebGL GPU rendering replaces fo-dicom WriteableBitmap |
| Image loading | **@cornerstonejs/dicom-image-loader 2.x** | Handles WADO-URI and WADO-RS; Web Worker pixel decoding; streaming prefetch |
| Tools | **@cornerstonejs/tools 2.x** | WindowLevel, Pan, Zoom, StackScroll, Flip — all real, replacing the WPF stubs |
| Volume loading | **@cornerstonejs/streaming-image-volume-loader** | MPR via VolumeViewport (new feature) |
| DICOM metadata | **dcmjs** | DICOM JSON normalization; `DicomMetaDictionary` for tag lookups |
| CSS | **CSS Modules** | Scoped per component; no runtime overhead |
| Serving | **Integrate with existing `dicomweb-viewer/server.js`** | Already sets COOP/COEP/CORP headers (required for SharedArrayBuffer/MPR), generates dynamic `app-config.js`, serves SPA with `index.html` fallback — zero server changes needed |

## 3. WADO Feature Additions (What's New vs WPF)

The WPF app only uses WADO-URI (download full `.dcm` to disk, open locally). React adds:

| Feature | WPF Approach | React Addition |
|---|---|---|
| **WADO-RS frames endpoint** | Not used | `GET /rs/studies/{}/series/{}/instances/{}/frames/1` — pixel data only, no DICOM header overhead. Called by cornerstone3D's `wadors:` loader via Web Worker |
| **Streaming prefetch** | Sequential foreach download (blocks render) | `imageRetrievalPool` concurrent fetch; first frame renders while rest load in background |
| **No disk write** | Every instance saved as `.dcm` on disk | Pixel bytes flow: network → Web Worker → GPU texture → canvas. No I/O |
| **Interactive Window/Level** | `//TODO WL/WW adjust by Mouse Drag` (unimplemented) | `WindowLevelTool` left-click drag; GPU shader update at 60fps; live WL/WW in corner overlay |
| **Pan, Zoom, Flip H/V, Invert** | Menu stubs (no commands) | Real cornerstone3D tools in a ToolGroup |
| **Stack scrolling via mouse wheel** | Slider only | `StackScrollMouseWheelTool` in addition to slider |
| **MPR (Volume rendering)** | Not present | 3-panel Axial/Coronal/Sagittal via `VolumeViewport` + `SharedArrayBuffer` (already enabled by `dicomweb-viewer/server.js` CORP/COEP headers) |
| **WADO-RS metadata** | Not used | `GET /rs/studies/{}/series/{}/instances/{}/metadata` — load DICOM tags without pixel data for instant overlay text |

## 5. Data Flow — MVVM+Messenger → React+Zustand

```
WPF:  Tree click → FileExplorerViewModel.Command
        → DicomFile.Open(localPath) [disk]
        → Messenger.Send(RenderSeriesMessage, "key_selectedChange")
          → Viewer2DViewModel.OnSelectedChange receives
            → DicomFile.Open(path) → DicomImage.RenderImage()
              → MainImage = WriteableBitmap

React: Tree click → studyTreeStore.selectInstance(sopUid)
         → studyTreeStore builds imageIds[] from QIDO-RS instance list
         → viewerStore.setActiveStack(imageIds, index)
           → CornerstoneViewport useEffect detects imageIds change
             → stackViewport.setStack(imageIds, 0)
               → dicom-image-loader Web Worker
                 → GET /rs/.../frames/1 (WADO-RS)
                   → decode pixels → GPU texture → WebGL render
               → metadata cache → viewerStore.updateOverlayText()
                 → CornerOverlay re-renders
```

Key replacement: `GalaSoft.MvvmLight.Messaging` string-keyed events → Zustand `store.getState().action()` direct typed calls.

## 8. What is NOT Migrated and Why

| WPF Feature | Decision | Reason |
|---|---|---|
| `ExitViewModel` | **Removed** | Browsers cannot close a tab programmatically |
| DIMSE C-Find / C-Get | **Removed** | Browsers have no raw TCP access; already replaced by QIDO-RS/WADO in `Legacy-viewer-Trans` |
| Local file OS dialog with arbitrary path | **Partial** | `<input type="file">` works but can't access arbitrary paths; user must pick each time |
| Recursive folder scan | **Partial** | `webkitdirectory` attribute covers the folder case; no background scanning |
| Annotation `.dcm` write to local disk | **Removed** | Browsers cannot write arbitrary files; would need STOW-RS on `dicomweb-api` as a future addition |
| WL/WW stubs, Zoom/Flip/Invert stubs | **Fully implemented** | These were TODO stubs in WPF; `@cornerstonejs/tools` makes them real in React |
