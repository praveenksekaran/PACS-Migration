## E2E Tests (Playwright)

### Prerequisites

The Vite dev server must be running before executing E2E tests.
Playwright starts it automatically via `webServer` in `playwright.config.ts`, but starting it manually avoids cold-start timeouts:

```bash
# Terminal 1 — start the dev server
cd C:\Praveen\Projects\dicomweb-pacs-src\dicomweb-react-viewer
npm run dev
```

### Run Commands

```bash
# Terminal 2
cd C:\Praveen\Projects\dicomweb-pacs-src\dicomweb-react-viewer

# Run all 111 E2E tests (headless Chromium)
npm run test:e2e

# Interactive UI mode — recommended for debugging
# Lets you click individual tests, see the browser live, inspect failures step-by-step
npm run test:e2e:ui

# Show HTML report after a completed run
npm run test:e2e:report

# Run a specific spec file
npx playwright test e2e/tests/01-startup.spec.ts

# Run tests matching a name (partial match)
npx playwright test --grep "reads apiUrl"

# Run with visible browser (headed mode)
npx playwright test --headed

# Skip live-server @api tests (run mock-only tests)
npx playwright test --grep-invert "@api"

# Run only live API tests against the deployed AWS server
API_SERVER_URL=http://3.27.26.186:5001 npx playwright test --grep "@api"
```

### Recommended Workflow

1. Start dev server in Terminal 1: `npm run dev`
2. Open UI mode in Terminal 2: `npm run test:e2e:ui`
3. Click individual tests to run/debug interactively
